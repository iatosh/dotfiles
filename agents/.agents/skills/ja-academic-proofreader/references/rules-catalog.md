# 添削ルールカタログ

`refs/paper-tips.md`（論文Tips）と `refs/japanese-tips.md`（日本語Tips）から抽出した全ルール。
日本語添削Skill構築のための基盤資料。（レビュー反映版）

## 検出方法の凡例

- **S-hard**: textlint / 形態素解析 / スクリプトで確実に検出。**error** として指摘。
- **S-soft**: 機械検出できるが正用も多い。**warning** 止まり。LLM層が文脈精査・修正案を付す。
- **H (Hybrid)**: スクリプトで候補抽出 ＋ LLMで最終判定。LLM層が有効なときのみ出力。
- **L (LLM)**: 文脈・意味・論理構造に依存。LLM層が有効なときのみ出力。

`[要設定]` = 文書種別（予稿／学位論文・ジャーナル・国際会議／研究計画書）などの設定値が必要。

---

## 実行モデル（起動時設定と出力レベル）

スキル起動時に確認する：

1. **LLM検出を使うか（on/off）**
2. **（.tex検出時）文書種別** — H-11（本稿／本論文／本研究）の判定用

出力構成：

- **静的層（常時実行）**: textlint ＋ 形態素解析（kuromoji）。LaTeXルールは `.tex` 検出時のみ有効化（両対応・自動判別）。
  - S-hard → **error**
  - S-soft → **warning**（連用形・受動態・がある 等。正用もあるため warning まで）
  - 実装上の例外: **S-17/S-18/S-19/S-21** は検出が発見的（段落・見出し・数式末尾の推定）で誤検出を避けるため `warning` で出力する（分類上は S-hard）。
  - severity は中央表で持たず**各ルールの定義場所**に置く（フレーズ系=prh-academic.yml の各エントリ、その他=各 lib の検出箇所）。ルールIDは参考情報として出力するのみ。
- **LLM層（LLM検出=on のときのみ）**: H系・L系 ＋ S-soft候補の文脈精査・修正案。
  - **LLM検出=off** のときは H系・L系を出力しない（静的層のみ）。
  - **Claude Code等のコーディングアシスタント内で使う場合**、外部LLM呼び出しはせず、ホストのエージェント自身が SKILL.md のルーブリックに従って文脈判定を行う。

### A-2. S-soft（warning）

| ID | ルール | 検出パターン例 | 出典 |
|----|--------|----------------|------|
| S-05 | **連用形の接続**（「〜し」「〜であり」等）に警告（S-06・L-10を統合） | 連用形接続（kuromoji） | jp L66,76,82 |
| S-07 | 受動態を極力用いない | `〜される` / `〜された` / `〜られる` | jp L164 |
| S-10 | 「〜がある」(there is/are) を避ける | 文末 `がある。` | paper L80 |
| S-12 | 「〜が提案／報告されている」を避ける | `が提案されている` / `が報告されている` | paper L84 |
| S-13 | 「解析する／調査する」の濫用 | `を解析` / `を調査` | paper L135,140 |
| S-23 | 「Pができれば、Qができる」型（十分条件の誤用） | `できれば、〜できる` | jp L158 |

> **S-05（連用形・統合ルール）**: 連用形（「〜し」「〜であり」等）は前後の節の接続関係（時系列／因果／並列）を曖昧にする。
> - 連用形自体に warning を付す（理想は明示的接続：「〜した後」「処理後のSに対して」等）。
> - ただし2文に分けると**同一主語が連続**する場合は連用形可。その際はB節を主語・述語文にし、Aと主語を変える（A and B型。例：「てんかんは〜神経疾患であり、〜悪影響を及ぼす**ことが知られている**」）。
> - LLM層が「接続が時系列/因果/並列のどれか」「修正方針（明示的接続 or 主語を変えた主述文）」を判定。
> - 受動態S-07は「既存定義への言及」では正用（L-24参照）。
