## P1: アブストラクト

### P1-1: アブストラクトの構成

**チェック**: アブストラクトが序論の要約ではなく、論文全体の要約になっているか？

**正しい構成**: 背景 → 目的 → 方法 → 結果 → 結論

**違反の兆候**: 背景説明が半分以上を占めている

**Nature式の構成**:
1. 1〜2文：基本的な導入（分野の背景）
2. 背景と動機をより詳細に
3. "Here we show…" で始まる一般的な問題の提示
4. 結果の要約
5. 結果の一般的な解釈と意義を1〜2文で

---

### P1-2: 「しかしながら」の使用

**原則**: 「しかしながら（However）」は切り札として1度だけ使う

---

## P2: 序論

### P2-1: 序論の流れ

**正しい流れ**:
1. **第一段落**: 誰もがわかる話題から具体的な技術課題へ（3〜5文，広い間口）
2. **第二段落**: その技術課題をどのような技術で解決できるか
3. **第三段落**: これまでどのような技術が生まれてきたか、それらの不足点は何か（However〜）
4. **第四段落**: 本論文では、どのような仮定で、どのような問題を設定し、どう解くか

**原則**: 大きな話題→小さな話題，過去→未来，ロジックは直線的

---

### P2-2: 記述の区別

| 種類 | 例 |
|------|-----|
| 事実 | 「2023年現在，日本における自動車の普及率は〇〇%である [文献]」 |
| 過去の知見 | 「Tanakaらは，△△であることを，fMRIの計測によって示した [文献]」 |
| 自分の考え | 「以上のことから，〜ではないかと仮説を立てた」 |

**原則**: 自分の考えは事実・知見を積み上げた上で、「以上のことから」のように明示して述べる

---

### P2-3: 「しかしながら」の使用

**原則**: 「しかしながら（However）」は切り札として**1度だけ**使う。ただし**必ず入れる**。

**隠れしかしながら文に注意**: 「しかし」がなくてもロジックとして逆接の文がある場合、それも1回とカウント

---

### P2-4: 悪魔の証明

**避けるべき表現**:
- 「〜は明らかになっていない」
- 「〜の研究はない」
- 「〜は調査されていない」

**問題**: 存在しないことは証明できない

```
✗ 人間の脳がどのように〇〇を処理しているのかは未だに明らかになっていない．

○ 人間の脳における〇〇の処理機構を，△△の観点から定量的に評価する手法が
  求められている．
```

**注意**: 「誰もやっていない」ことは新規性として十分ではない

---

### P2-5: 「〜がある」構文

**問題**: 前文との関連が切れる、根拠薄弱・天下り感

```
✗ 未知システムに対して，それをブラックボックス関数とみなしてシステムを
  最適化する手法の一つとしてベイズ最適化がある．

○ ベイズ最適化は，未知システムに対して，それをブラックボックス関数とみなして
  システムを最適化する手法の一つである．
```

---

### P2-6: 曖昧な導入

| 避けるべき表現 | 問題 | 代替 |
|--------------|------|------|
| 近年 | 何年前か不明 | 具体的な年（2020年以降など） |
| 〜の分野では | 分野の切り取り方は人それぞれ | 具体的な応用・技術名で記述 |

---

### P2-7: 「〇〇では」構文

**問題**: 「先行研究では」「実験では」などと突然書き出すと、何のことか分からない

```
✗ 今日はいい天気だ．学校では，1時間目に国語がある．

○ 今日はいい天気だ．朝から気持ちよく登校できそうだ．学校では，1時間目に国語がある．
```

---

### P2-8: 課題設定

**避けるべき表現**:
- 「〇〇率向上の余地がある」
- 「実用化には至っていない」
- 「性能がまだ低い」

**問題**: 当たり前のことであり、技術的課題が具体的でない

---

## P3: 方法

### P3-1: 図の引用位置

```
✗ 傾聴タスクでは...想像タスクでは...発声タスクでは...
  各タスク時の呈示画面を図1に示す．

○ 図1に示すような画面を呈示することで，傾聴，想像，発声の3種類のタスクを
  実験参加者に課した．傾聴タスクでは...
```

**原則**: 図は説明の前か同時に引用

---

### P3-2: 具体的な記述

**問題**: 「解析する」「調査する」がワイルドカードとして濫用される

```
✗ 脳部位ごとに6種類の電極グループに分けて解析した．
○ 脳部位ごとに6種類の電極グループに分けてフレーズ誤り率を評価した．
```

---

## P4: 結果

### P4-1: 結果の記述

**必須要素**:
1. 図の種類（箱ひげ図，折れ線グラフ等）
2. 軸・データ点が何を表すか
3. 図から読み取れる傾向の文章化
4. 考察に繋げられる記述

```
✗ 3年生の各教科の成績を図1〜図5に示す．

○ 3年生の5教科について，性別で条件づけした成績分布（箱ひげ図）を図1〜5に示す．
  図1に示すように，国語の平均点は男子**点，女子**点であり，
  統計的有意差は認められなかった（t=**, p=**, d=**）．
```

**避けるべき書き方**: 図1〜図5を同じ構造で羅列する（間違い探しになる）

---

### P4-2: 図表の言及

```
✗ 表xxより，全ての実験参加者においてtap-stabilityの値は0.80以上となった．
  → 「表xxが原因で値が0.80以上になった」と読める

○ 表xxに示すように，全ての実験参加者においてtap-stabilityの値は0.80以上となった．
```

---

### P4-3: 比較級の使用

```
✗ 5つの分類器で，テストデータ正解率が高くなった
  → 何と比べて？「高くなった」は状態変化を示す

○ 5つの分類器で，テストデータ正解率がそれ以外の区間より
  無音呈示直前区間において高かった
```

---

### P4-4: 母集団の明記

```
✗ 馴染みありと馴染みなしそれぞれの群におけるdPTEの平均

○ 馴染みありと馴染みなしそれぞれの群内で被験者について平均したdPTE
```

---

### P4-5: 統計的検定

**必須**: 統計量・p値・効果量をセットで示す

```
○ 統計的有意差は認められなかった（t=2.34, p=0.12, d=0.45）
```

---

## P5: 考察

### P5-1: 考察の構成

1. 結果の要約（この後の考察の伏線になるように）
2. 仮説・想定との照らし合わせ
3. 参考文献を引用した議論

---

### P5-2: 文献との対話

**注意**:
- 先行研究との一致/不一致にフォーカスしすぎない
- 「答え合わせ」ではなく、自分の結果から何が言えるかを議論

---

## P6: 体裁

### P6-1: 図のフォーマット

**使用すべき**: ベクタ形式（pdf, eps）
**禁止**: ビットマップ形式（png, jpg, bmp）

**注意**: 既存のPNGをPDFに変換するのはNG。Pythonなら`savefig()`でベクタ形式を直接出力。

---

### P6-2: 図の配置

**LaTeX**: `\begin{figure}[t]`（tオプション）を使用

---

### P6-3: 数式

**原則**: 数式は文章の一部として書く。文末にはピリオドを付ける。

```
○ 2つの統計量A，Bに対して，その平均は
       μ = (A+B)/2                    (1)
   で定義される．
```

**能動態/受動態**:
- 能動態：自分が定義する場合
- 受動態：既存の定義に言及する場合

**禁止**: 数式を定義前に引用する

---

### P6-4: 参考文献

**推奨**: IEEE方式（電気情報系の標準）
