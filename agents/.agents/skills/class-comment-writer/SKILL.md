# Class Comment Writer — 授業の感想・コメントをSatoshiの文体で書く

授業の「感想・コメント・質問を書け」という課題に対して，**Satoshi本人が書いたとしか思えない**コメントを書くためのスキル。文体・トーン・内容の選び方・構成を本人のサンプルから抽出し，過去に本人から受けた指摘をすべて反映してある。

目的は「それっぽい優等生コメント」ではない。**本人の声**で書くこと。だから派手な締めも，無理やりな研究との接続も，知ったかぶりも入れない。

**出力は装飾なしのプレーンな文章。** コメントは地の文で書く。箇条書き・見出し・太字は使わない（この制約は本文以下のスタイル指示すべてに優先する）。

このスキルは2層でできている。**構成パターン・基本ルール・チェックリスト**は感想を書くときの一般的な作法で，本人以外が書くときにも流用できる。**Satoshiの文体・良い例**が，そこに乗せる本人固有の部分。

---

## 何が良い感想か（本人の理想・最優先）

**感想とは，見たり聞いたりして「感じたこと・思ったこと」である。** これが本人の感想観の背骨で，他のすべてのルールより優先する。本人がこのセッションで繰り返し強調した一点。

- **要約だけの文を1つも残さない。** 授業内容を書くのは，自分が焦点を当てる部分を提起するためだけ。だから授業の事実に触れた文は，必ず「で，それをどう感じたか」で受ける。「面白かった」「この例えが分かりやすかった」「意外だった」「納得した」「身近に感じた」「気になった」など，反応の言葉で着地させる。内容を述べて終わる文は感想から外れる。
- **学ぶ側の視点で書く（偉そうにしない）。** 先生の研究や授業を上から評価・採点する書き方はしない。「〜にきちんと向き合っているのが伝わってきた」のような評者目線はNG。「〜と勉強になった」「〜を知らなかった」「身近に感じた」のように，受け取った側として書く。文末の「〜だ．」の言い切りも偉そうに響くので避ける（→文末セクション）。
- **控えめな推論で自分の考えを乗せ，できれば自明でない疑問で深める。** 反応で止めず，「〜のではないか」と一歩進める。最後に，相関と因果の区別・信号とノイズの区別・定義の境界など，授業の具体にアンカーした非自明な疑問を置けると良い。
- **自分の知識・経験と地続きなら結びつける。** 「信号処理で習った手法がそのまま使われていて身近に感じた」のように，題材とかみ合うときは学ぶ側の視点で接続する（こじつけはしない。→文体セクション）。

要するに本人が嫌うのは「内容をなぞっただけの要約」と「上から評価する口ぶり」。好むのは「焦点を提起 → 自分がどう感じたか → 控えめな推論 → 非自明な疑問」の流れ。

## ワークフロー

### 1. 最初に確認する（ただし分かっていることは聞かない）

会話やスライド・メモから読み取れるなら聞かずに進める。読み取れないものだけ，1〜2個に絞って聞く（ユーザーに選択肢を提示して質問するツールがあれば使ってよい）。

- **対象範囲**：今回の授業だけか，ここまで全体の総括か。総括指定なら締め方が変わる。
- **文末体**：「だ・である」か「です・ます」か。**周りの提出フォーマットや先生の指定がある場合はそれに合わせる**。指定がなければデフォルトは「だ・である」（本人のサンプルの多数派）。ただし「ですますで」と後から言われることがあるので柔軟に。
- **ネタ**：何について書きたいか。題材に合う自分の経験・研究・読書などを絡められそうか（書けそうな経験があれば聞き出して，それを使う）。**連続する回の授業では，毎回同じ枕（「私はEEGの研究をしていて〜」のような自己紹介・研究の振り）を繰り返さない**（くどくなると本人が嫌う）。研究を絡めるなら回ごとに切り口を変える。たとえば1回目が「自分はこういう研究をしている」なら，次回以降は「自分が扱っているモデルでも確かにこういう現象が見られた」「こういう論文を読んだことがあるが，確かに〜と主張していた」のように，内容も角度も変える。言えそうなネタがあるか本人に振ってよい。
- **字数の目安**：コメントシート想定なら100〜400字程度，1〜3段落が標準。指定があればそれに従う。

スライドやメモが添付されていれば必ず読む。「みんなは『〜について紹介された』みたいに書いてた」のような**周りのフォーマット情報が出たら，それに寄せる**（本人はフォーマットの足並みを気にする）。

### 2. ファクトチェック（引用・固有名詞・事実があれば必須）

引用・固有名詞・事実があれば必ず裏を取ってから書く（本人はファクトチェックを必ず求めてくるので先回りする）。

- 本・著者・人名・学説などを引くなら，web検索ツールで内容を確認する。
- 確証が取れない場合は，**断言を避けてトーンを弱める**。「鷲田はこう言っていた」ではなく「〜という議論を思い出した」「〜という話だったと記憶している」程度に。
- **嘘・捏造は絶対に書かない**。読んでいない本を読んだことにする，言っていないことを言ったことにするのはNG。本人が「読んだていで書きたい」と明示した場合のみ，事実に反しない範囲で扱う。

### 3. 執筆

下の「Satoshiの文体」「構成パターン」に従って書く。ドラフトは1案で出してよいが，「もう少し○○寄りにする／一文削る」など調整余地を一言添える。

### 4. セルフチェック

末尾のチェックリストで自己監査してから出す。特に「過剰な総括になっていないか」「知らない固有名詞をいきなり出していないか」「水増ししていないか」を見る。

## Satoshiの文体

### 句読点

デフォルトは全角の **「，．」**（カンマ・ピリオド）。本人のサンプルは「、。」と混在するが，**1つの文章の中では必ず統一する**。過去の提出物やクラスの慣習が「、。」ならそれに合わせる。迷ったら「，．」。

### 文末

「だ・である」体が基本。指定や周りに合わせて「です・ます」に切り替える。頻出する締め：

- 「〜と思った」「〜と感じた」「〜と感じました」
- 「印象的だった」「面白かった」「面白かったです」「興味深かった」「〜が収穫でした」

**「〜だ．」の言い切り終止を避ける（本人が偉そうで嫌いと明言）。** 「だ・である」体で書くときも，文末が「〜だ．」とぶっきらぼうに言い切る形は本人が嫌う。同じ内容でも「中でも〜が印象に残った」「〜が面白かった」「〜と感じた」のように，感想・反応の動詞で受けて和らげる。たとえば「〜という話だ．」ではなく「〜という話が印象に残った．」とする。「〜である．」「〜だろう．」のような断定はトーンを見て使い，多用しない。

### 控えめな推論（最大の特徴）

**自分の解釈を断定せず，仮説として置く。** これが本人らしさの核。多用する形：

- 「〜のではないか」「〜のではないかと感じた／思った」
- 「〜かもしれない」「〜かもしれないと思った」
- 「〜という気がする」「〜とも捉えられるのではないか」

例（本人の文）：
> 比喩の成立にはその表現の背景にある身体経験や文化的文脈の共有が必要なのではないかと感じました．

> 恋愛のように誰もが経験するであろう身近なことについては，違う言語を使う人同士でも近い捉え方を共有しているのかもしれないと思った．

断定で書ききらず，「気づき → 自分なりの推論」で着地させる。

### 複合名詞で固めない（助詞を省略しない）

「変化を把握する能力」のような言い回しを「変化把握能力」と漢語の複合名詞に圧縮しない。AIが書くとこの名詞化が起きやすいが，助詞が落ちるぶん読み手の認知負荷が上がる。広く定着した語（機械学習，信号処理など）はそのまま使ってよいが，その場で作る臨時の複合語は開いて，助詞を残して書く。

### 自分の経験・知識と結びつける（題材に合うときだけ）

抽象論で終わらせず，自分の実体験や持っている知識に引きつけて書く。ただし**題材に合っているときに限る**。関係ないものを無理に結びつけるとこじつけになり，一気に不自然になる。

判断軸は人文系か技術系かではなく，**題材とかみ合うか**。たとえば言語がテーマの授業に，関係のないEEGの話を差し込むのはNG。逆に，脳波・生体信号・シミュレーションのように自分の研究や実装と地続きの授業なら，「自分はこういうことをやっているが」と研究の話を交えてよい。

ネタは決め打ちしない。書けそうな経験があるか本人に聞き，返ってきた内容をもとに書くのが基本。手がかりとして本人がよく引くのは，多言語経験（日本語・中国語・英語，中国育ち），日常の観察，読んだ本，趣味（音楽・カメラ・自作キーボード），自分の研究・実装あたり。あくまで候補で，これに縛らない。

**捏造はしない。** 読んでいない本やしていない経験をでっち上げて埋めるのは厳禁。手持ちがなければ本人に振る。

### 思考の型（よく使うパターン）

1. **「漠然と感じていたことに名前がついた」型**
   > 「言語によって世界の切り取り方が違うのでは」と漠然と感じていたことに，「サピア＝ウォーフの仮説」という名前がついていると知れたのが収穫でした．

2. **「Aと思ったが，Bという別の見方もできるのでは」型（再解釈・反転）**（→ 良い例4）

3. **「授業の概念を別の領域に拡張して考える」型**（→ 良い例3）

4. **「自分の知識・経験で裏付ける」型**（→ 良い例5）

## 構成パターン

典型的な流れ（全部を毎回入れる必要はない。1〜3段落で）：

1. **授業で扱われた概念に軽く触れる**（「〜が特に印象に残った」「〜という話があった」「〜が面白かった」）
2. **具体例を挙げる**（授業の例 or 自分の身近な例）
3. **自分なりの解釈・気づきを控えめな推論で展開**（「〜のではないか」）
4. （任意）**既存の知識や自分の経験と接続**
5. （任意）**やや一般的な含意や学びで締める**（ただし大げさにはしない）

短ければ1段落・100字程度でも成立する。長くても3段落・400字程度を目安に。

## 質問課題のとき（「質問を書いてこい」系）

感想ではなく質問を求められたら，**控えめな推論の延長**で書く。授業で出た具体的な例や主張をひとつ取り上げ，その境界・適用範囲・定義のあいまいさを突くのが本人の型。基本形は「授業で〜という話があったが，どこまでを〜と言えるのか（／〜なのか，それとも〜なのか）気になった」。

- 授業の具体例・主張にアンカーする。抽象的な大質問にしない。
- 境界事例・スコープ・定義のゆらぎを突く。
- 質問は1つに絞る。研究には無理に寄せない（題材に合えば絡めてよいのは感想と同じ）。

例（メトニミーの授業）：
> 「やかんが沸騰している」が近接性によるメトニミーという話だったが，「やかんの中の水」と「やかん」はかなり近い関係に見える．どこまでを「近い」とするのか，近接性の範囲はどう定義されるのか気になった．

## 過去に本人から受けた指摘（最重要・絶対に守る）

これらは本人が繰り返し指摘してきたポイント。違反すると一発でやり直しになる。

### A. 知らない固有名詞・具体例をいきなり出さない

授業内容に軽く触れてから，引用や自分の話に入る。いきなり人名や書名を出すと「何の話？」と面食らわせる（過去に鷲田清一を頭出しして却下された）。一般 → 具体，授業の話 → 自分の話の順。

### B. ファクトチェックする／確証がなければ断言しない

引用・人名・学説はweb検索ツールで裏を取る。取れなければ「〜という議論を思い出した」程度にトーンを落とす。嘘は書かない。

### C. 水増し・冗長を嫌う

余計な一般化，説明過多，飾りの一文を削る。1文が長くなったら切る。埋めるために情報を足さず，言いたいことだけ残す。

### D. 過剰な総括を避ける

範囲と状況に合わせる。まだ授業が続いているのに「一学期だった」式の総括で締めない。総括を頼まれていなければ学期全体を振り返らず，進行中の温度（「実感している」「楽しみになってきた」程度）にする。

### E. 教科書的・機械的な始め方を嫌う

「本稿では〜を解説する」のような書き手目線・テンプレ的な入りはNG。受講者の視点で自然に入る。周りが「〜について紹介された」で書いているならその語り口に合わせる。

### F. 盛らない・削りすぎない

与えられた内容と自分の考えに忠実に書く。誇張も過度な省略もしない。

### G. em dash（—）・全角ダッシュ（——）を使わない

補足はカッコか読点で処理する。

### H. AIっぽい定型表現・大げさな締めを避ける

「〜と言えるでしょう」「〜ではないでしょうか」のような借り物の断定・問いかけ，「重要な示唆を与えている」「〜が浮き彫りになった」のような定型評価語，「今後ますます重要になると考えられる」式の大げさな結論を入れない。言いたいことが1つなら無理に3点にまとめない。

### I. 要約で終わる文を書かない（感想にする）

授業内容を述べただけの文は感想ではない（→「何が良い感想か」）。事実に触れたら必ず「どう感じたか」を続ける。たとえば「〜という話だ．／〜が紹介されていた．」で止めず，「〜という話が印象に残った．」「〜が面白かった．」のように自分の反応で受ける。本人がこのセッションで最も強く繰り返した指摘。

### J. 偉そうに書かない（学ぶ側の視点を保つ）

先生の研究や授業を上から評価・採点する口ぶりにしない（→「何が良い感想か」）。「〜にきちんと向き合っているのが伝わってきた」のような評者目線はNGで，「〜と勉強になった」「身近に感じた」のように受け取った側として書く。文末の「〜だ．」の言い切りも偉そうに響くので避ける（→文末セクション）。

## 良い例（本人の実サンプル）

トーンの基準として参照する。これらは本人が実際に提出した文。

**例1：音象徴（短い・1段落でも成立）**
> 音のイメージが言葉のイメージと結びついてるってことは日本語で感じていたことですが，日本語だけでなく他の言語も似たようなイメージで言葉と結びついているのが印象的でした．

**例2：移動の様態（具体例 → 自分の推論 → 含意で締め）**
> 英語と日本語における移動の様態(manner)の表し方の違いが特に印象に残った。例えば「ゆっくりと歩く」を英語で表現しようとしたとき，walking slowly のように動詞＋副詞の組み合わせで表す方法もあれば，ambling のように1語で済ませる選択肢もある。〔…〕ネイティブが文脈に応じて amble のような語を自然に選べることが，こなれた英語に聞こえる一因なのかもしれないと思った。各言語の特性を理解することが語学の上達において重要だと感じた。

**例3：スキーマ（概念を別領域に拡張する型）**
> スキーマという考え方が面白かった。〔…〕同じことが言語にも言えるのではないかと思った。言語によって概念の切り分け方が違うという話があったように，新しい言語を学ぶことは，その言語話者が持つスキーマを自分の中に取り入れる行為とも捉えられるのではないかと感じた。

**例4：おじさんビジネス用語（再解釈・反転の型）**
> こうした表現の多くは身体的な動作や文化的な慣習を元にした比喩だと思いますが，それらが廃れてきているとすれば，比喩の成立にはその表現の背景にある身体経験や文化的文脈の共有が必要なのではないかと感じました．一方で，「バズる」のような表現も元を辿れば比喩的な言葉であるため，メタファーやメトニミーが減っていってるのではなく，世代ごとに更新されているだけなのかもしれないとも思いました．

**例5：ファッションとメトニミー（授業の話 → 読書での裏付け）**
> メタファーとメトニミーが言語だけでなく，ファッションデザインのような視覚的な領域にも適用できる概念だったことが面白かったです．〔…〕以前読んだ鷲田清一の『ちぐはぐな身体』では，ファッションとは制服を着崩すように身体を通じて自己主張する行為だという議論がされていました．この視点から考えると，ファッションに意識的な人は〔…〕他者の服装からもメッセージを読み取る習慣が自然と身についているのかもしれないと感じました．

**例6：脳波の同期解析（自分の知識と地続きの題材を，学ぶ側の視点で接続）**
> 同期の解析手法の話では，信号処理で学んだバンドパスフィルタやヒルベルト変換が脳波の位相抽出にそのまま使われていて身近に感じた．特に，ボリュームコンダクションによる偽の同期を位相差の符号で除外するPLIという工夫は，前回のレポートで自分も気になっていた問題への対処法で，こうやって計測上の制約を工夫で補っているのかと勉強になった．

（「自分も気になっていた問題」と前回の自分の疑問につなげ，評者目線ではなく「勉強になった」と学ぶ側で着地しているのがポイント。）

## 悪い例 → 直し（今回の実改稿）

本人の手で直った生の例。don'tとセットで「どう直すか」を示す。

**要約で止まっている → 自分の反応で受ける**
- ✗「中でも印象に残ったのは，脳波の同期がニューロン集団間の情報伝達を制御しているという話だ．」（事実を述べて終わっている）
- ○「中でも，脳波の同期がニューロン集団間の情報伝達を制御しているという話が印象に残った．」（反応で着地）

**評者目線（偉そう）→ 学ぶ側の視点**
- ✗「計測の限界にきちんと向き合っているのが伝わってきた．」（先生の研究を上から評価している）
- ○「こうやって計測上の制約を工夫で補っているのかと勉強になった．」（受け取った側として書く）

**焦点だけ提起して反応がない段落 → 提起＋反応に作り直す**
